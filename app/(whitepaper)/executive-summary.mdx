### Core Concept
ParaGame Protocol is a decentralized research framework that explores algorithmic fairness in blockchain-based applications through:

- **Mathematical Fairness**: Provably fair lottery mechanics with transparent odds
- **Blockchain Transparency**: All operations recorded on PRIZM blockchain
- **Cryptographic Security**: PLRNG ensures unpredictable but verifiable randomness

### Technical Foundation

- **Epoch System**: 1,550 block cycles (~24 hours) 
- **Randomness**: PLRNG with multi-layer entropy
- **Distribution**: Multi-tier prize allocation algorithm
- **Integration**: Native PZM token usage only

## ðŸŽ® How ParaGame Works

### 1. Epoch Registration Phase
- **Duration**: Variable length, up to 1,550 blocks
- **Action**: Participants select 6 numbers (1-36) and stake PZM tokens
- **Storage**: All combinations stored immutably on blockchain
- **Validation**: Smart contracts verify valid combinations and stakes

### 2. Random Number Generation Phase  
- **Trigger**: Automatic at epoch completion
- **Method**: PLRNG uses multi-layer entropy sources:
  - Current blockchain state (hashes, timestamps)
  - Previous epoch outcomes
  - Network participation patterns
  - Cryptographic randomness functions
- **Output**: 6 numbers (1-36) generated deterministically but unpredictably

### 3. Prize Distribution Phase
- **Algorithm**: Multi-tier matching system
  - **Exact Order Match**: 90% of prize pool
  - **Any Order Match (6 numbers)**: 50% of remaining pool
  - **5 Number Match**: 25% of remaining pool
  - **4 Number Match**: 15% of remaining pool  
  - **3 Number Match**: 5% of remaining pool
  - **2 Number Match**: 2% of remaining pool
- **Automation**: Smart contracts automatically distribute winnings
- **Rollover**: Unclaimed funds transfer to next epoch
